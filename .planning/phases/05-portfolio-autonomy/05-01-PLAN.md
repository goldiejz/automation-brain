    Create `scripts/ark-portfolio-decide.sh` with the following exact structure:

    1. Shebang `#!/usr/bin/env bash`. Header comment block describing the AOS Phase 5
       portfolio priority engine, the 4 public functions, the priority formula
       `priority = stuckness * 3 + falling_health * 2 + (monthly_headroom > 20 ? 1 : 0) + ceo_priority * 5`,
       and the 4 decision classes (SELECTED, DEFERRED_BUDGET, DEFERRED_HEALTHY,
       NO_CANDIDATE_AVAILABLE).

    2. NO `set -euo pipefail` at top level — this is a sourced library.

    3. Resolve `VAULT_PATH="${ARK_HOME:-$HOME/vaults/ark}"` and
       `ARK_PORTFOLIO_ROOT="${ARK_PORTFOLIO_ROOT:-$HOME/code}"`.

    4. Source `ark-policy.sh` with graceful degradation (mirror lines 28-37 of
       `scripts/ark-policy.sh` itself):
       ```bash
       _PORTFOLIO_LIB_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" 2>/dev/null && pwd)"
       if [[ -f "$_PORTFOLIO_LIB_DIR/ark-policy.sh" ]]; then
         # shellcheck disable=SC1091
         source "$_PORTFOLIO_LIB_DIR/ark-policy.sh"
       else
         _policy_log() { echo "stub:_policy_log $*"; }   # test-friendly stub
         policy_config_get() { echo "$2"; }
         policy_config_has() { return 1; }
       fi
       ```

    5. **`portfolio_scan_candidates [root]`** — Echoes one project path per line.
       - Default root = `$ARK_PORTFOLIO_ROOT`.
       - `find "$root" -maxdepth 3 -type f -name STATE.md -path '*/.planning/STATE.md'`
       - For each match, echo `dirname(dirname($STATE_md))` (i.e., the project root).
       - Sort + uniq the output.
       - Return 0 even if zero matches.

    6. **`_portfolio_read_customer <project_path>`** (private helper) — Echoes the
       customer slug or `scratch`.
       - Reuse the awk parser from `policy-config.sh::_pc_read_yaml_key` (copy the
         same awk block; do NOT source policy-config.sh's private helpers — they're
         underscore-prefixed). Read `bootstrap.customer` from
         `<project>/.planning/policy.yml`.
       - If empty → echo `scratch`.

    7. **`_portfolio_stuckness <project_path>`** (private helper) — Echoes 0|1|2.
       - Read `<project>/.planning/STATE.md`. If grep finds `status: blocked` →
         echo 2. Else if last-modified timestamp of STATE.md is older than 7 days
         (use `stat -f %m` on macOS; fallback `stat -c %Y` for Linux test runners):
         echo 1. Else echo 0.

    8. **`_portfolio_falling_health <project_path>`** (private helper) — Echoes 0|1.
       - Look for `<project>/.planning/delivery-logs/*.log` newest file. Grep for
         `pass count` or `tests passed` and compare last-2 numeric occurrences. If
         decreasing → echo 1; else 0. If insufficient history → echo 0.
       - Conservative: any awk/grep failure → 0.

    9. **`_portfolio_mtime <project_path>`** (private helper) — Echoes Unix epoch
       seconds of `<project>/.planning/STATE.md` (BSD `stat -f %m` first; GNU
       `stat -c %Y` fallback).

    10. **`portfolio_score_project <project_path>`** — Echoes ONE TSV line:
        `<path>\t<customer>\t<phase>\t<stuckness>\t<falling_health>\t<budget_headroom>\t<ceo_priority>\t<total>`
        - `phase` = `grep -oE 'Phase [0-9]+' STATE.md | head -1` → `?` if none.
        - `budget_headroom` = stub returns `100` (Plan 05-02 replaces in
          SECTION:budget-reader).
        - `ceo_priority` = stub returns `0` (Plan 05-03 replaces in
          SECTION:ceo-directive).
        - `total` = `stuckness * 3 + falling_health * 2 + (budget_headroom > 20 ? 1 : 0) + ceo_priority * 5`.
        - Use bash arithmetic `$(( ... ))`. No floats.

    11. **`portfolio_pick_winner [root]`** — Orchestrator.
        - Calls `portfolio_scan_candidates` → for each path, call
          `portfolio_score_project`. Capture all TSV rows.
        - Filter: skip rows where field 6 (`budget_headroom`) ≤ 0 (Plan 05-02 uses
          this to express DEFERRED_BUDGET).
        - Sort numerically descending by field 8 (`total`); tie-break by
          `_portfolio_mtime` descending (most-recently-touched).
        - Echo the winning project path on stdout. If zero candidates → echo
          empty + return 0 (NOT non-zero; caller decides escalation).

    12. **`portfolio_decide [root]`** — Full flow.
        - Stub for Plan 05-04: scaffold ONLY. Calls `portfolio_pick_winner`,
          captures result, and emits a placeholder
          `_policy_log "portfolio" "SELECTED" "<reason>" "<context_json>"` call
          with TSV breakdown serialized to JSON. Echoes the winning project path.
        - If empty winner → `_policy_log "portfolio" "NO_CANDIDATE_AVAILABLE" "no projects under $root"` and echo nothing.
        - **Plan 05-04 expands** decision classes (DEFERRED_BUDGET, DEFERRED_HEALTHY) and 24h cool-down inside SECTION:audit-and-cooldown.

    13. **Append three empty sentinel sections at the END of the file**, before the
        self-test, in this exact order:

        ```bash
        # === END SECTION: budget-reader ===

        ```
        These sentinels are LOAD-BEARING for Wave-2 parallelism. Do not remove.

        - Create `$TMP_BASE/vault` and `$TMP_BASE/portfolio/{proj-a,proj-b,proj-c}/.planning/STATE.md`

          timestamp — bash 3 compat: `date -v -10d +%Y%m%d0000` on macOS,

          policy.yml with `bootstrap.customer: acme`.
        - Project C: `Phase 1`, mtime = now-2d. policy.yml with
          `bootstrap.customer: beta`. STATE.md contains `status: blocked`.

        - Override `_policy_log`/`policy_config_get` shims via the stub fallback
