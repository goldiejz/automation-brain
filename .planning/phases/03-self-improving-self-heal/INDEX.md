## Wave-3 file-conflict note

(03-03 adds `learner_apply_pending` + lock helpers; 03-04 adds `learner_write_digest`),
so logically they're independent. Practically, the executor must run them sequentially
(03-03 first, then 03-04) to avoid edit conflicts. The depends_on field reflects logical
dependency only (both depend on 03-02); the executor uses plan-number order to serialize
the writes within Wave 3.

## Phase 2 → Phase 3 contract (locked)

Phase 3 consumes the Phase 2 audit-log schema verbatim. Specifically:

- `schema_version` stays at `1` — Phase 3 does NOT bump.
- `outcome` is patched in place via jq, keyed on `decision_id`.
- `correlation_id` is used to chain `class:self_improve` audit entries back to the source decision_ids that drove the learning.
- `class:self_improve` is a NEW class value (used only by auto-patcher audit entries) — adding a value is not a schema migration.
- True-blocker classes (`escalation`, `budget/ESCALATE_MONTHLY_CAP`) are NEVER auto-promoted/deprecated. The filter is applied in two places (emission in 03-02, re-check at apply-time in 03-03) for defense in depth.

## Phase 2 lessons honored (avoid regression)

From Phase 2's iter-3 revision (validated by 3 checker rounds):

- **Single audit writer:** All audit log writes go through `_policy_log` from `ark-policy.sh`. 03-03 sources ark-policy.sh and calls `_policy_log "self_improve" ...`; never inline-writes to policy-decisions.jsonl. Mirrors NEW-B-2.
- **Bash 3 compat (macOS):** No `declare -A`, no `${var,,}`, single-quoted heredocs for Python. Self-tests grep for these on the lib files.
- **Isolated test vaults:** Tier 9 follows NEW-W-1 — `mktemp -d`, copy scripts, run against ARK_HOME=$TMP_VAULT, never touch the real `policy-decisions.jsonl`.
- **No `read -p` in delivery path:** 03-05 includes a regression check (`grep -nE 'read[[:space:]]+-p' scripts/ark-deliver.sh* | grep -v '# AOS: intentional gate'` returns 0).
- **Concurrency safety:** `mkdir`-lock in 03-03 (macOS doesn't ship `flock` by default; `mkdir` is atomic on POSIX filesystems).
