## AOS Phase 6.5 — CEO Dashboard Contract

**Locked decisions (D-DASH-*):**
- D-DASH-TIER-A-FIRST: Bash version ships before Rust TUI; phase exit gate requires both.
- D-DASH-READONLY: Dashboard never writes to policy.db, ESCALATIONS.md, or any vault file. Mark-resolved delegates to `ark escalations --resolve` (existing single-writer).
- D-DASH-INVOCATION: `ark dashboard` (no flag) → Tier A bash. `ark dashboard --tui` → Tier B Rust binary.
- D-DASH-RUST-DEPS: ratatui + rusqlite + crossterm only. No async runtime.
- D-DASH-RUST-BUILD: `cd scripts/ark-dashboard && cargo build --release` → `scripts/ark-dashboard/target/release/ark-dashboard`.
- D-DASH-REFRESH: Rust TUI polls every 2s via `crossterm::event::poll(500ms)` + 2s elapsed-since-last-tick check.
- D-DASH-DRIFT-TOLERANCE: Drift detector treats STATE.md vs disk diffs within 60s as INFO (not RED).
- D-DASH-DEGRADE: Color-friendly terminal degradation: tput colors < 8 OR NO_COLOR=1 → plain output.

**Sections (locked priority order, both tiers):**
1. Portfolio grid · 2. Escalations panel · 3. Budget summary · 4. Recent decisions · 5. Learning watch · 6. Drift detector · 7. Tier health.

**Read-only invariant enforcement:**
- Bash: every `sqlite3` call uses `-readonly` flag.
- Rust: `Connection::open_with_flags(..., SQLITE_OPEN_READ_ONLY | SQLITE_OPEN_NO_MUTEX)`.
- Tier 13.3 + Tier 13.9-11 capture real-vault md5 before + after every dashboard invocation; assertion that md5 unchanged.

**Out of scope (Phase 8 candidates):** web dashboard, push notifications, multi-machine sync, historical trend charting, employees plugin UI.
